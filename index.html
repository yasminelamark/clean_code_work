Você sabia que a manutenção de um software é tão importante quanto a sua construção? Mas para mantê-lo, é importante adotar práticas de Clean Code (Código Limpo) no time de desenvolvimento. Portanto, o Clean Code é mais do que uma metodologia, é um conceito que é muito adotado nas equipes de tecnologia de empresas e startups para deixar o software o mais longe possível dos riscos de bugs e em dia. Veja, portanto, neste artigo os principais princípios, as boas práticas e as vantagens de trabalhar com o foco no código limpo.

Como surgiu? O Clean Code e as suas técnicas foram apresentadas pela primeira vez no livro “Clean Code: a Handbook of Agile Software Craftsmanship”. O livro foi escrito por Robert Cecil Martin (conhecido como Uncle Bob) e lançado em 2008. Martin é conhecido na área de desenvolvimento desde 1970. Ele é um dos 17 signatários do Manifesto Ágil, escrito em 2001. Seu livro sobre Código Limpo se tornou um best seller na área. Hoje ele é consultor e escritor de livros que abordam o tema.
